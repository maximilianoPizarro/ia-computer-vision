= CRD reference

Helm templates under `charts/all/` are the source of truth. This guide shows simplified examples with links to official product documentation.

== How a self-service OIDC client becomes a rate-limit tier

When a workshop user runs the OIDC credentials self-service scaffolder template in Developer Hub, the chosen plan tier flows through the gateway stack in five steps:

. The user selects `planTier` (`free` or `gold`) on the template form.
. The template creates a Keycloak client with an `oidc-hardcoded-claim-mapper` protocol mapper that embeds `plan: <tier>` in every JWT.
. The client obtains a token from realm `cv` (`client_credentials` or Bearer flow).
. Kuadrant `AuthPolicy` (`authpolicy-cv` on `neuroface-cv-lb`) validates the JWT issuer (`keycloak.apps.<hub-domain>/realms/cv`) and admits the request.
. `RateLimitPolicy` and `PlanPolicy` on `neuroface-cv-lb` match `auth.identity.plan == "free"` or `"gold"` and apply `/v1/predict` limits (100/h or 500/h).

The MaaS route (`ai-maas`) uses the same dual-auth pattern (`APIKEY` or OIDC JWT) with `PlanPolicy` and `TokenRateLimitPolicy` tiers on `/v1/chat/completions`.

=== OIDCPolicy

`extensions.kuadrant.io/v1alpha1` — browser redirect OIDC login on Gateway API HTTPRoutes (interactive sessions).

[NOTE]
====
`neuroface-cv-lb` deliberately does not use `OIDCPolicy`. That route serves API clients with Bearer tokens, not browser logins. An `OIDCPolicy` redirect breaks CORS preflight (OPTIONS requests cannot carry a token). Realm `cv` on `neuroface-cv-lb` uses `AuthPolicy` + `jwt-users` instead (see below).
====

[source,yaml]
----
apiVersion: extensions.kuadrant.io/v1alpha1
kind: OIDCPolicy
metadata:
  name: oidc-neuroface
  namespace: neuroface-gateway-system
spec:
  provider:
    issuerURL: https://sso.apps.<hub-domain>/realms/neuroface
    clientID: client-neuroface-user1
    authorizationEndpoint: https://sso.apps.<hub-domain>/realms/neuroface/protocol/openid-connect/auth
    tokenEndpoint: https://sso.apps.<hub-domain>/realms/neuroface/protocol/openid-connect/token
    redirectURI: https://neuroface.apps.<hub-domain>/auth/callback
  targetRef:
    group: gateway.networking.k8s.io
    kind: HTTPRoute
    name: neuroface-app-lb
  auth:
    tokenSource:
      authorizationHeader:
        prefix: Bearer
        name: Authorization
      cookie:
        name: jwt
----

*Connection*: OIDCPolicy → HTTPRoute → Gateway → backend Service (Skupper listeners on hub reach spoke connectors). Per-user scaffolded NeuroFace instances on spokes use the same redirect pattern against their local RHBK realm.

Chart: `charts/all/neuroface-gateway/templates/oidcpolicy-realms.yaml`

=== AuthPolicy

`kuadrant.io/v1` — API key and/or JWT authentication. A single `AuthPolicy` can declare multiple
named identity sources under `rules.authentication`; Authorino evaluates all of them and admits
the request if *any one* succeeds. The AI Gateway route (`ai-maas`) uses this to accept either a
Kuadrant API key or a Keycloak realm-`maas` OIDC token on the same endpoint ("dual auth").

[source,yaml]
----
apiVersion: kuadrant.io/v1
kind: AuthPolicy
metadata:
  name: ai-maas-auth
  namespace: ai-gateway-system
spec:
  targetRef:
    group: gateway.networking.k8s.io
    kind: HTTPRoute
    name: ai-maas
  when:
    - predicate: request.method != "OPTIONS"
  rules:
    authentication:
      api-key-users:
        apiKey:
          allNamespaces: true
          selector:
            matchLabels:
              app: workshop-llm-tokens
              kuadrant.io/apikey: "true"
        credentials:
          authorizationHeader:
            prefix: APIKEY
      jwt-users:
        jwt:
          issuerUrl: https://keycloak.apps.<hub-domain>/realms/maas
----

*Connection*: AuthPolicy → HTTPRoute → APIProduct (Developer Hub catalog) → API key Secret, or → Keycloak realm (OIDC JWT).

Chart: `charts/all/workshop-kuadrant-apis/templates/policies.yaml`

=== RateLimitPolicy

`kuadrant.io/v1` — per-route request limits keyed off authenticated identity (JWT claims or client IP).

[source,yaml]
----
apiVersion: kuadrant.io/v1
kind: RateLimitPolicy
metadata:
  name: neuroface-cv-ratelimit
  namespace: neuroface-gateway-system
spec:
  targetRef:
    group: gateway.networking.k8s.io
    kind: HTTPRoute
    name: neuroface-cv-lb
  limits:
    predict-free:
      when:
        - predicate: 'request.path.matches("^/v1/predict")'
        - predicate: 'has(auth.identity) && has(auth.identity.plan) && auth.identity.plan == "free"'
      counters:
        - expression: auth.identity.sub
      rates:
        - limit: 100
          window: 1h
    predict-gold:
      when:
        - predicate: 'request.path.matches("^/v1/predict")'
        - predicate: 'has(auth.identity) && has(auth.identity.plan) && auth.identity.plan == "gold"'
      counters:
        - expression: auth.identity.sub
      rates:
        - limit: 500
          window: 1h
    predict-default:
      when:
        - predicate: 'request.path.matches("^/v1/predict")'
        - predicate: '!(has(auth.identity) && has(auth.identity.plan))'
      counters:
        - expression: request.headers['x-forwarded-for']
      rates:
        - limit: 30
          window: 1m
----

*Connection*: RateLimitPolicy → HTTPRoute. The `plan` JWT claim (from the self-service template `planTier` parameter) selects the `predict-free` or `predict-gold` tier.

Chart: `charts/all/neuroface-gateway/templates/ratelimit-policy.yaml`

=== PlanPolicy

`extensions.kuadrant.io/v1alpha1` — publish discoverable rate-limit tiers in the Kuadrant DevPortal plugin (Developer Hub API entity Kuadrant tab).

[source,yaml]
----
apiVersion: extensions.kuadrant.io/v1alpha1
kind: PlanPolicy
metadata:
  name: neuroface-cv-plans
  namespace: neuroface-gateway-system
spec:
  targetRef:
    group: gateway.networking.k8s.io
    kind: HTTPRoute
    name: neuroface-cv-lb
  plans:
    - tier: free
      predicate: |
        has(auth.identity) && has(auth.identity.plan) && auth.identity.plan == "free"
      limits:
        custom:
          - limit: 100
            window: 1h
    - tier: gold
      predicate: |
        has(auth.identity) && has(auth.identity.plan) && auth.identity.plan == "gold"
      limits:
        custom:
          - limit: 500
            window: 1h
----

*Connection*: PlanPolicy → HTTPRoute → DevPortal discovered-plans UI. Pairs with RateLimitPolicy on the same route.

Chart: `charts/all/neuroface-gateway/templates/plan-policy.yaml`

=== TokenRateLimitPolicy

`kuadrant.io/v1alpha1` — token-budget rate limits for LLM endpoints (counts tokens, not requests).

[source,yaml]
----
apiVersion: kuadrant.io/v1alpha1
kind: TokenRateLimitPolicy
metadata:
  name: ai-maas-token-limits
  namespace: ai-gateway-system
spec:
  targetRef:
    group: gateway.networking.k8s.io
    kind: HTTPRoute
    name: ai-maas
  limits:
    free:
      rates:
        - limit: 50
          window: 1m
      when:
        - predicate: request.path == "/v1/chat/completions"
        - predicate: |
            auth.identity.metadata.annotations["secret.kuadrant.io/plan-id"] == "free" ||
            auth.identity.groups.split(",").exists(g, g == "free")
      counters:
        - expression: auth.identity.userid
    oidc:
      rates:
        - limit: 50
          window: 1m
      when:
        - predicate: request.path == "/v1/chat/completions"
        - predicate: has(auth.identity.iss)
      counters:
        - expression: auth.identity.sub
----

*Connection*: TokenRateLimitPolicy → HTTPRoute (`ai-maas`). Works with dual-auth AuthPolicy and PlanPolicy tiers (`free`, `gold`, `oidc`).

Chart: `charts/all/workshop-kuadrant-apis/templates/policies.yaml`

=== APIProduct

`devportal.kuadrant.io/v1alpha1` — API catalog entry for the Kuadrant Developer Hub plugin.

[IMPORTANT]
====
`metadata.name` must exactly match the Backstage API entity name. The Kuadrant frontend plugin builds "View in Catalog" links as `/catalog/default/api/<metadata.name>`, not from `documentation.docsURL`. Example: `neuroface-cv-openapi` (not `neuroface-cv` or `neuroface-cv-oidc`).
====

[source,yaml]
----
apiVersion: devportal.kuadrant.io/v1alpha1
kind: APIProduct
metadata:
  name: neuroface-cv-openapi
  namespace: neuroface-gateway-system
spec:
  displayName: Computer Vision API
  description: YOLO PPE detection API protected by realm-cv OIDC JWTs.
  publishStatus: Published
  approvalMode: manual
  version: v1
  tags: [neuroface, cv, oidc, ppe, edge, gateway-api]
  documentation:
    docsURL: https://developer-hub.apps.<hub-domain>/catalog/default/api/neuroface-cv-openapi
  contact:
    email: platform@example.com
  targetRef:
    group: gateway.networking.k8s.io
    kind: HTTPRoute
    name: neuroface-cv-lb
----

Chart: `charts/all/neuroface-gateway/templates/apiproducts.yaml`

=== KeycloakRealmImport

[source,yaml]
----
apiVersion: k8s.keycloak.org/v2alpha1
kind: KeycloakRealmImport
metadata:
  name: realm-cv
  namespace: keycloak-system
spec:
  keycloakCRName: keycloak
  placeholders:
    CLIENT_SECRET_USER1:
      secret:
        name: keycloak-client-cv-user1
        key: clientSecret
  realm:
    realm: cv
    enabled: true
    roles:
      realm:
        - name: manage-clients
          description: Can create and manage own OIDC clients
    users:
      - username: user1
        clientRoles:
          realm-management:
            - view-clients
            - manage-clients
----

*Connection*: KeycloakRealmImport → Keycloak CR → ExternalSecret (Vault client secrets).

=== ZTunnel

`sailoperator.io/v1` — deploys the ambient mesh data plane (`ztunnel` DaemonSet). Required for L4 mTLS and Istio metrics (`istio_requests_total`, `istio_tcp_*`) consumed by Kiali and Grafana.

[source,yaml]
----
apiVersion: sailoperator.io/v1
kind: ZTunnel
metadata:
  name: default
  namespace: istio-system
spec:
  namespace: ztunnel
  version: v1.28.8
----

*Connection*: ZTunnel → `ztunnel` namespace DaemonSet on every node. Without this CR, `oc get istio default -n istio-system` reports `ZTunnelNotFound` and mesh telemetry panels show no data.

Chart: `charts/all/istio-ztunnel/templates/ztunnel.yaml`
