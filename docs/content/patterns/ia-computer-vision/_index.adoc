== What it does

This pattern solves the challenge of running *secure, observable AI inference at the edge* while maintaining centralized control through GitOps.

++++
<div class="vp-features">
  <div class="vp-feature">
    <div class="vp-icon">&#x1F916;</div>
    <h3>NeuroFace + YOLO Inference</h3>
    <p>Real-time PPE detection via OVMS serving YOLO models on OpenShift AI. Each spoke cluster runs an independent inference pipeline.</p>
  </div>
  <div class="vp-feature">
    <div class="vp-icon">&#x1F310;</div>
    <h3>Multi-cluster Gateway</h3>
    <p>RHCL (Kuadrant) gateway on the hub load-balances inference traffic 50/50 across east and west spokes via Skupper tunnels.</p>
  </div>
  <div class="vp-feature">
    <div class="vp-icon">&#x1F512;</div>
    <h3>Zero-trust Security</h3>
    <p>Istio ambient mesh with mTLS, RHACS vulnerability scanning, Kuadrant AuthPolicy with OIDC, and Vault-managed secrets via ESO.</p>
  </div>
  <div class="vp-feature">
    <div class="vp-icon">&#x1F4CA;</div>
    <h3>Full-stack Observability</h3>
    <p>OpenTelemetry collectors, Thanos multi-cluster federation, Grafana dashboards, and Kiali service mesh visualization.</p>
  </div>
  <div class="vp-feature">
    <div class="vp-icon">&#x1F6E0;</div>
    <h3>Developer Self-service</h3>
    <p>Red Hat Developer Hub scaffolds per-user NeuroFace instances with RHBK, HTTPRoute, and Tekton pipelines on spokes.</p>
  </div>
  <div class="vp-feature">
    <div class="vp-icon">&#x1F504;</div>
    <h3>Fully GitOps</h3>
    <p>ArgoCD + RHACM drive all 3 clusters from a single Git repo. Spoke registration, Skupper linking, and operator installs are automated.</p>
  </div>
</div>
++++

== Industry use cases

* **Manufacturing** — PPE compliance monitoring on factory floors with real-time alerts via Kafka and Mailpit
* **Retail** — In-store inventory and safety monitoring across geographically distributed locations
* **Energy** — Remote site safety compliance with edge inference and centralized oversight
