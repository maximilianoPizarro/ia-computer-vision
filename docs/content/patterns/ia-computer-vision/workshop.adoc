= Workshop mode

The AI Computer Vision pattern includes workshop mode for hands-on events. Workshop mode is **enabled by default** with **30 users** (`user1` through `user30`).

== What gets provisioned

[cols="2,3,2",options="header"]
|===
| Platform | Access method | Default password

| OCP Console
| HTPasswd identity provider `workshop-users`
| `Welcome123!`

| Developer Hub
| Keycloak realm `backstage` (OIDC)
| `Welcome123!`

| GitLab
| User account + `ws-userN` group (owner)
| `Welcome123!`

| DevSpaces
| Git credentials in `userN-devspaces` namespace
| `Welcome123!`
|===

Special accounts: `platformadmin` (platform engineering group) and `admin` (workshop administrator).

== Module 02 — Scaffold NeuroFace with RHBK biometric OIDC

Showroom module 02 walks through the AI Computer Vision software template journey:

. Log in to Developer Hub as `userN` / `Welcome123!`
. Open **Create** → **AI Computer Vision at the Edge**
. Select target spoke (east or west) and confirm hub/spoke apps domains
. Submit — the scaffolder creates a GitLab repo in `ws-workshop/neuroface-userN`
. Argo CD ApplicationSet on the **hub** (`vp-gitops`) discovers the repo and deploys `k8s/` manifests to the selected spoke
. Verify ApplicationSet: `oc get applicationset -n vp-gitops user-neuroface-apps`

For operator interaction diagrams and expected results, see link:../scaffolding-and-secrets/[Scaffolding and secrets].
. Open your catalog entity → tab **API** → NeuroFace OpenAPI spec with **Try It Out**
. Click **Open in DevSpaces** to edit NeuroFace source
. Tab **CI** shows Tekton pipeline run status (when Tekton plugin is enabled)
. Access NeuroFace: `https://neuroface-spoke-gateway.<spoke_domain>/user/userN/`
. RHCL redirects to RHBK OIDC — login `userN` / `Welcome123!`
. First login: complete biometric enrollment (webcam, 3–5 images)
. Subsequent logins: password + facial 2FA verification
. Test PPE detection and MaaS chat in the NeuroFace UI
. Trigger a PPE violation (remove hardhat in webcam) — NeuroFace publishes to Kafka topic `cv.ppe.detections`
. Open **Mailpit** from the OCP console menu → verify alert email arrived
. Verify Kafka consumer: `oc logs -n mailpit deploy/ppe-kafka-mailpit -f`

== Scaling workshop users

Update `userCount` consistently in these application overrides:

* `platform-users`
* `developer-hub`
* `gitlab-operator`
* `openshift-ai-hub`
* `devspaces`
* `showroom` (`showroom.terminal.userCount`)
